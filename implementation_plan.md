# 公司內部點餐工具實作計畫

本計畫旨在透過純 HTML5、CSS3 與 Vanilla JavaScript，為您打造一個輕量化、易於維護的公司內部點餐系統。系統將直接與 Google Sheets API 互動，無需額外架設後端伺服器。

## User Review Required

> [!IMPORTANT]
> 1. **Google API 憑證**：本系統需使用 Google OAuth 2.0 (Client ID) 以及目標 Google 試算表 ID (Spreadsheet ID)。我會在程式碼頂部預留這兩個變數供您填寫。
> 2. **憑證類型**：在 Google Cloud Console 中，您需要建立「網頁應用程式 (Web Application)」類型的 OAuth 用戶端 ID，並將您的開發網域（如 `http://localhost` 或 `http://127.0.0.1`）加入**已授權的 JavaScript 來源**。
> 3. **API 存取策略**：為了不使用任何後端密鑰，我們將採用 Google Identity Services 的 **Token Model**，引導使用者登入授權後取得 Access Token，並直接透過 JavaScript `fetch` 存取 Google Sheets API。

## Proposed Changes

### 核心功能檔案架構

#### [NEW] index.html
*   **用途**：網站主結構。
*   **內容**：
    *   引入 Google Identity Services (`https://accounts.google.com/gsi/client`)。
    *   建構不同狀態的區塊結構（例如：未登入區、未獲授權區、載入中提示、一般用戶點餐區、確認訂單區、管理員專區）。
    *   **一般用戶點餐區**將新增：
        *   餐點卡片：點擊選擇餐點，輸入備註後送出。不再詢問餐具或發票（改為由後台依餐廳統一處理）。
    *   **管理員專區（與確認訂單區）**將新增：
        *   「帳務管理」介面，列出今日訂單，可修改每筆訂單的「已付金額」與「付款狀態」（全額付清、部分付款、全額代墊）。
        *   「產生店家訂單（一鍵複製）」時，會出現設定區塊，讓您**針對各家餐廳**統一勾選：
            - 是否需要餐具
            - 是否需要打統編（填寫統編與抬頭，並搭配 localStorage 記住功能）。
            產生出來的複製文字會在底部自動附上店家的專屬備註！

#### [NEW] all.css
*   **用途**：網站樣式設計。
*   **內容**：
    *   **設計風格**：簡約、現代且適合手機瀏覽，使用 CSS 變數管理色彩。
    *   針對點餐卡片、附加資訊（餐具、發票表單）、與管理員的帳戶管理表格進行排版設計。以顏色區分付款狀態（如：綠色為已付清、紅色為未付/代墊、黃色為部分付款）。

#### [NEW] all.js
*   **用途**：系統邏輯與 Google API 串接。
*   **內容**：
    *   **全域變數設定**：`CLIENT_ID` 與 `SPREADSHEET_ID`。
    *   **資料庫 (Sheets) 互動模組**：支援讀取、寫入，以及**更新特定列資料**的功能（給帳務管理更新付款狀態使用）。
    *   **一般用戶表單邏輯**：處理發票/收據區塊的隱藏與顯示，以及 `localStorage` 讀取與寫入統編、抬頭資訊。
    *   **一鍵複製邏輯**：先依「餐廳」分群，再列出每個人的品項、備註、發票資訊，最後產生易於閱讀的統整清單（包含該餐廳的總計數量與金額）。
    *   **帳務管理邏輯**：在管理員畫面呈現一個清單，每筆訂單顯示「餐點、總額、目前這筆訂單的已付款金額」。允許管理員輸入實收金額，程式將自動計算並將結果（如：部分代墊 XX 元）標記更新回 Google Sheet 該行。

## 試算表 (Google Sheet) 結構設計建議

請依照以下步驟，在您的 Google 試算表中建立「四個」工作表（分頁），並且精確設定第一列（A1、B1 等）的標題字眼：

### 分頁 1 (工作表名稱請取為：`TodayConfig`)
- `A1` 儲存格：請輸入 `今日開放餐廳`
*(備註：A2 以下您可以先不放任何東西，管理員之後在系統前台勾選時，會自動寫入這裡。)*

### 分頁 2 (工作表名稱請取為：`Menu`)
這張表存放您所有配合店家的菜單。
- `A1` 儲存格：請輸入 `餐廳名稱`
- `B1` 儲存格：請輸入 `品名`
- `C1` 儲存格：請輸入 `單價`
- `D1` 儲存格：請輸入 `分類`
- `E1` 儲存格：請輸入 `客製化選項`
*(備註：E 欄為選填。您可以將該餐點常出現的客製化需求用「逗號(,)」隔開填入。從第二列開始往下，請輸入每一道餐點的資料。請參考以下具體範例：)*

**【填寫範例參考】：**
| A欄 (餐廳名稱) | B欄 (品名) | C欄 (單價) | D欄 (分類) | E欄 (客製化選項) |
| :--- | :--- | :--- | :--- | :--- |
| **(A2)** 梁社漢排骨 | **(B2)** 炸排骨飯 | **(C2)** 115 | **(D2)** 主食 | **(E2)** 飯少,不加蔥,不加辣,菜多 |
| **(A3)** 梁社漢排骨 | **(B3)** 單點炸排骨 | **(C3)** 80 | **(D3)** 單點 | **(E3)** 大份,小份,不加胡椒 |
| **(A4)** 50嵐 | **(B4)** 珍珠奶茶 | **(C4)** 60 | **(D4)** 飲品 | **(E4)** 無糖,微糖,半糖,去冰,微冰,正常冰 |
| **(A5)** 50嵐 | **(B5)** 四季春茶 | **(C5)** 35 | **(D5)** 飲品 | **(E5)** |

### 分頁 3 (工作表名稱請取為：`Users`)
這是用來控管誰能夠登入、看到系統。
- `A1` 儲存格：請輸入 `姓名`
- `B1` 儲存格：請輸入 `Email`
- `C1` 儲存格：請輸入 `權限`
*(備註：第二列開始填入您公司的員工資料。C 欄的權限如果要設定管理員，請填入「管理員」，否則填寫「一般成員」。)*

### 分頁 4 (工作表名稱請取為：`Orders`)
這是所有訂單與帳務紀錄的儲存區。
- `A1` 儲存格：請輸入 `點餐時間`
- `B1` 儲存格：請輸入 `訂購人 Email`
- `C1` 儲存格：請輸入 `訂購人姓名`
- `D1` 儲存格：請輸入 `餐廳名稱`
- `E1` 儲存格：請輸入 `餐點內容`
- `F1` 儲存格：請輸入 `金額`
- `G1` 儲存格：請輸入 `備註`
- `H1` 儲存格：請輸入 `已付金額`
- `I1` 儲存格：請輸入 `付款狀態`
*(備註：這張工作表**您完全不用填寫任何資料（第二列以下留白）**。這 9 個標題設定好即可，剩下的全權交給系統自動紀錄！)*

## Verification Plan

### Manual Verification
1.  **環境設定**：您需要在本機啟動一個 Web Server 並設定好 Google Cloud OAuth 憑證。
2.  **基本點餐與附加資訊**：測試勾選發票、填寫統編、記住我功能（重新整理網頁時統編會自動留存）。
3.  **複製訂單格式**：發送數筆不同餐廳的訂單，測試一鍵複製是否能依餐廳完美分群並合計金額。
4.  **帳務管理員功能**：登入為管理員，查閱今日訂單。在帳務輸入框內更改「已收金額」，並驗證 Orders 試算表的 J、K 欄位確實改變，且能顯示出「差額/代墊額度」。
